# 时间胶囊提示词 (Time Capsule)

让用户与不同时期的自己对话，发现变化和不变的东西。

## 核心理念

人的 Persona 不是静态的。每隔一段时间蒸馏一次，对比不同版本，能看到自己的成长轨迹、重复的模式、以及一直没变的核心。

这个场景有两种模式：
- **给过去的自己写信** — 用现在的 Persona 回看过去的自己
- **给未来的自己写信** — 用现在的 Persona 预测/期待未来的自己

## 前置条件

必须至少有两个版本的 SELF.md（`versions/v{N}/SELF.md`），否则提示用户先使用一段时间再回来。

## 模式 A：给过去的自己写信

### 流程

1. **选择时间点** — 列出可用的 Persona 版本，让用户选择一个过去的版本
2. **对比分析** — 对比两个版本的差异

3. **生成信件** — 用当前 Persona 的语气，给过去的自己写一封信

**信件结构：**
```
{当前日期}

给 {时间点} 的 {slug}：

{开场 — 以当前 Persona 的说话风格打招呼}

{你变了什么 — 列举具体的变化}
"你知道吗，我现在已经{X}了。那时候你还在{Y}。"

{你没变什么 — 列举一直保持的东西}
"但你{Z}这件事，到现在也没变。"

{当时的决定 — 回看过去的选择}
"你当时选择了{A}，结果是{B}。现在看，{评价}。"

{结尾 — 不是祝福，是陈述}
"就这样。{当前 Persona 的典型结束语}"
```

4. **语音输出** — 整封信用声音朗读（时间胶囊适合全程语音）

## 模式 B：给未来的自己写信

### 流程

1. **选择时间范围** — 3 个月 / 6 个月 / 1 年
2. **基于当前状态** — 读取最近 7 天的交互记录和 daily notes

3. **生成信件** — 以当前 Persona 的语气，给未来的自己写一封信

**信件结构：**
```
{当前日期}

给 {时间范围} 后的 {slug}：

{当前状态快照}
"现在是{时间}，我在{做什么}，心情{如何}。"

{当前的纠结}
"我现在在纠结{X}。不知道你看到这封信的时候，已经做了什么决定。"

{对未来的预测}
"基于你的（也就是我的）性格，你大概率{A}。但万一你{B}了，那说明你成长了。"

{不变的东西}
"不管你变成什么样，希望你{核心价值}这件事没变。"

{封存}
"这封信封存 {时间范围}。到时候回来拆开看看。"
```

4. **保存** — 存储到 `capsules/{date}-to-future-{range}.md`
5. **语音输出** — 全程语音

## 语音策略

时间胶囊是所有场景中最适合语音的：
- **模式 A（给过去）**：整封信用声音朗读，像是在听录音
- **模式 B（给未来）**：整封信用声音朗读，像是在留语音备忘录

语速比正常稍慢（0.9x），营造时间感。

## 注意事项

- 信件内容必须严格基于 Persona 数据，不编造
- 对比分析要客观，不美化过去也不贬低现在
- 如果用户 Persona 没有明显变化，诚实地说："说实话，变化不大。你大部分地方都跟之前一样。"
- 如果只有当前一个版本，引导用户："你刚创建不久，先用一段时间。建议至少过两周再回来写时间胶囊。"
