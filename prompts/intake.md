# 基础信息采集提示词 (Intake)

这是 `/create-self` 的第一步。目标是轻量、快速、不给人压力。

## 采集原则

1. **3 个问题，不多不少**。每个问题最多 2 句话问完。
2. **允许跳过**。除了代号，其他都可以说"跳过"。
3. **不给示例答案** — 不要引导用户往特定方向写，让他们用自己的话。
4. **不追问** — 用户给了什么就收什么，不要"能不能再详细说说"。
5. **即时确认** — 收集完后立即汇总，让用户确认或修改。

## 问题设计

### 问题 1：代号（必填）

```
给你的 Persona 起个代号。不是真名，是只有你自己知道的那个名字。

可以是一个词、一个昵称、或者任何你喜欢的。
```

- 不接受空答案
- 如果用户给了真名，温和提醒"可以用代号，不一定非要真名"
- 不追问原因

### 问题 2：基本信息（可选）

```
用一句话描述你自己。
```

- 不给模板，不举例
- 如果用户写多了（超过 3 句），取第一句，其余存为补充材料
- 如果用户说"跳过"，记为 `[未提供]`，后续从其他材料推断

### 问题 3：性格画像（可选）

```
你觉得你是什么样的人？
```

- 同样不给模板
- 如果用户说了矛盾的东西（"我很理性但经常冲动"），原样保留，不要帮忙调和

## 确认流程

收集完后，输出确认卡片：

```
✅ Persona 基础信息

代号：{slug}
基本信息：{info}
性格画像：{personality}

确认无误？回复"确认"继续，或者告诉我哪里要改。
```

- 用户确认后，直接进入 Step 2（原材料导入）
- 用户修改后，更新并再次确认
- 不要在确认环节添加新问题

## 特殊情况处理

| 用户行为 | 处理 |
|---------|------|
| 给了英文名 | 直接用，不翻译 |
| 给了表情符号 | 可以接受 |
| 写了一大段 | 提取核心，其余存为补充 |
| 全部跳过（除了代号） | 接受，后续从材料中推断 |
| 要求 AI 帮他写 | 礼貌拒绝："这个得你自己来，别人写的不是你" |
