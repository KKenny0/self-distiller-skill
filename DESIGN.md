# 🪞 self-distiller — 设计文档

> 把你自己蒸馏成 AI，再用你自己的声音跟自己对话。

## 1. 愿景

**self-distiller 是一个自我反思工具，不是聊天机器人。**

核心体验：你跟一个了解你的、用你自己的声音说话的 AI 对话。不是让 AI 替你想，而是让你听到自己的想法后产生新的觉察。

心理学依据：研究表明，听到自己的声音能显著增强自我认知和情绪调节能力（PMC 11274574, 2024; ScienceDirect, 2022）。

## 3. 蒸馏层（Distill）

### 3.1 原材料采集

参考 ex-skill / colleague-skill 的采集模式，支持以下数据源：

| 数据源 | 格式 | 优先级 | 说明 |
|--------|------|--------|------|
| Discord 聊天记录 | OpenClaw 原生 | ⭐⭐⭐ | 已有基础设施，可直接读取 |
| OpenClaw MEMORY.md | Markdown | ⭐⭐⭐ | 自动积累，无需额外导入 |
| Daily notes | Markdown | ⭐⭐⭐ | 自动积累 |
| 微信聊天记录 | 导出格式 | ⭐⭐ | 需要用户手动导出 |
| Twitter/X 推文 | URL/JSON | ⭐⭐ | 已有 twitter-watch 基础设施 |
| 博客/文章 | URL/Markdown | ⭐⭐ | Content Parser 可用 |
| 语音样本 | 音频文件 | ⭐⭐⭐ | 用于 ListenHub Voice Clone |
| 邮件 | .eml/.mbox | ⭐ | 可选 |
| 照片/截图 | 图片 | ⭐ | 可选，用于生活场景分析 |

### 3.2 Persona 提炼

从原材料中提炼以下维度：

```
SELF.md 结构：
├── 基础信息（非必须，用户自愿提供）
│   ├── 称呼/昵称
│   ├── 年龄/职业/生活状态
│   └── 当前关注领域
├── 说话风格
│   ├── 常用语言（中文/英文/混合）
│   ├── 口头禅和高频表达
│   ├── 句式偏好（短句/长句/反问/感叹）
│   ├── 幽默风格（冷幽默/自嘲/梗）
│   └── 表达密度（简洁/详细/啰嗦）
├── 思维模式
│   ├── 决策风格（直觉型/分析型/混合）
│   ├── 价值观优先级
│   ├── 常见认知偏差（确认偏误/乐观偏差等）
│   ├── 典型论证模式
│   └── 压力下的反应模式
├── 知识边界
│   ├── 熟悉领域
│   ├── 不了解/不感兴趣的领域
│   └── 学习中的领域
├── 情感模式
│   ├── 常见情绪触发点
│   ├── 情绪表达方式（直接/内敛/爆发）
│   └── 情绪恢复模式
└── 行为模式
    ├── 日常工作节奏
    ├── 习惯性拖延/坚持
    ├── 社交偏好
    └── 信息获取偏好
```

### 3.3 蒸馏流程

**Step 1：基础信息录入（3 个问题）**

参考 ex-skill / colleague-skill 的 intake 模式，只问 3 个问题：
1. 花名/代号（必填）
2. 基本信息（一句话：职业、状态、关注什么）
3. 性格画像（一句话：MBTI、性格标签、思维风格）

**Step 2：原材料导入**

展示数据源选项，用户选择并提供：
```
原材料怎么提供？

[A] OpenClaw 自动采集（推荐）
    自动读取 MEMORY.md、daily notes、Discord 聊天记录

[B] 手动导入
    聊天记录（微信/QQ/Discord）
    社交媒体（Twitter/微博）
    写作/笔记（博客/Obsidian/Markdown）

[C] 混合模式
    先用 [A] 自动采集，再补充 [B] 的额外数据

[D] 最小启动
    先用已有数据快速生成，后续再补充
```

**Step 3：Persona 生成**

1. 解析所有原材料（parser 按数据源类型分发）
2. LLM 分析提炼 → 生成 SELF.md
3. 展示摘要给用户确认
4. 用户可以纠正/补充

### 3.4 语音克隆设置

**Step 1：采集语音样本**

要求用户提供 1-3 段音频（建议总时长 1-5 分钟）：
- 可以是日常对话录音
- 可以是语音消息
- 可以是任何自然说话的录音

**Step 2：调用 ListenHub Voice Clone API**

```python
# 伪代码
clone_response = listenhub.clone_voice(
    name="kenny",
    audio_files=["sample1.mp3", "sample2.mp3"],
    language="zh"
)
speaker_id = clone_response["speaker_id"]
# 保存到 SELF.md 的配置中
```

**Step 3：验证克隆效果**

生成一段测试语音让用户确认质量。

## 4. 镜像层（Mirror）— 场景设计

### 4.1 晨间独白（Daily Mirror）⭐ P0

**触发方式：**
- 用户主动发起：`/mirror` 或 "跟我聊聊"
- Cron 定时推送（可选）：每天早上 8:00

**流程：**
1. 读取昨天的 daily notes / 最近的对话记录
2. AI 用用户的语气开场："早上好。昨天你提到了 [X]，现在感觉怎么样？"
3. 用户自由表达（文字或语音）
4. AI 用用户的语气回应：
   - 不给建议，而是像用户自己会跟自己说的那样
   - 帮助梳理思绪，点出用户可能忽略的模式
   - 引导用户自己找到答案
5. 用 ListenHub TTS 生成语音（用户的克隆声音）
6. 最后输出一段「今日觉察」文字 + 语音摘要

**输出示例：**
```
🪞 今日觉察

你今天提到三件事，但反复回到 [X] 这个话题。
根据你过去处理类似情况的方式，你可能会 [Y]。
但你的核心价值观里 [Z] 排在第一位——这两个有冲突。

[语音版本]
```

**语音输出：** 关键的反思段落用用户的克隆声音朗读。

---

### 4.2 情绪旁观者（Emotion Observer）⭐ P0

**触发方式：**
- 用户主动发起：`/observe` 或 "我需要冷静一下"
- 情绪关键词检测（可选，后续迭代）

**流程：**
1. 用户倾诉/吐槽/抱怨，想说什么说什么
2. AI 分析输入，识别：
   - 表面情绪 vs 真实情绪
   - 重复出现的表达模式（如多次用"算了"、"无所谓"）
   - 隐藏的诉求
3. AI 用用户的语音回放情绪核心（cognitive defusion）：
   - "你现在真正在意的是 [X]，不是表面上的 [Y]"
   - "你用了 3 次'算了'，说明你其实没放下"
   - "根据你过去在类似情况下的行为模式，你现在最可能做的是 [Z]"
4. 不安慰、不评判、不给建议。只是像一面镜子。

**心理学原理：** 认知解离（cognitive defusion）——跟自己的想法拉开距离才能看清它。用自己的声音做认知解离效果显著优于用别人的声音（PMC 11274574）。

---

### 4.3 内心辩论（Inner Debate）⭐ P1

**触发方式：**
- 用户主动发起：`/debate [主题]` 或 "我纠结要不要..."

**流程：**
1. 用户描述一个决定/两难选择
2. AI 用 ListenHub Multi-Character Script 模式，同一个克隆声音扮演两个版本：
   - 🔴 「冲动版你」：基于你的直觉模式、过往冲动决策、短期满足倾向
   - 🔵 「理性版你」：基于你的分析框架、长期价值观、理性思考模式
3. 两边各说 3 轮
4. 用户当裁判，可以随时打断
5. 最后用用户的语音总结

**ListenHub Multi-Character Script 示例：**
```json
{
  "scripts": [
    {"content": "说实话，我觉得应该直接冲了，想太多没用。", "speakerId": "kenny"},
    {"content": "等一下，上次你也是这么说的，然后呢？", "speakerId": "kenny"},
    {"content": "上次不一样，这次情况不同...", "speakerId": "kenny"}
  ]
}
```

> 注意：两个角色用同一个 speakerId（用户克隆声音），通过文本内容区分角色。

---

### 4.4 时间胶囊（Time Capsule）⭐ P2

**触发方式：**
- 用户主动发起：`/capsule`
- 每季度自动生成（可选）

**流程：**
1. 对比当前 SELF.md 和 N 个月前的版本
2. 标记变化维度：
   - 思维模式的变化
   - 价值观的演进
   - 新出现的口头禅/表达习惯
   - 情绪模式的变化
   - 关注领域的迁移
3. 用用户的语音读一封「给过去/未来的自己的信」

**输出示例：**
```
📦 三个月时间胶囊 (2026.01 → 2026.04)

思维变化：
- 以前你倾向于 [X]，现在你更倾向 [Y]
- "算了" 的使用频率下降了 40%

新出现的模式：
- 开始关注 [Z] 这个领域
- 面对冲突时从回避转向直接沟通

[语音版本：给三个月前的自己的一封信]
```

## 5. 进化层（Evolve）

### 5.1 偏差修正

用户随时可以纠正 AI 的表现：
- "我不会这么说" → 记录偏差，更新 SELF.md
- "这个判断不准确" → 调整思维模式描述
- "我的语气不是这样的" → 更新说话风格
- "/update-self" → 进入修正模式

### 5.2 数据追加

用户可以随时提供新数据来丰富 Persona：
- "我有新的聊天记录" → 导入并重新分析
- "我最近写了一篇文章" → 提取风格和观点
- "我最近的思维方式变了" → 触发 Persona 更新

### 5.3 版本管理

```bash
selfs/
└── kenny/
    ├── SELF.md              # 当前 Persona
    ├── versions/
    │   ├── v1/SELF.md       # 初版
    │   ├── v2/SELF.md       # 第一次更新
    │   └── v3/SELF.md       # 第二次更新
    ├── config.yaml          # 语音克隆配置
    │   └── speaker_id: "kenny-zh"
    ├── interactions/        # 对话记录
    │   ├── 2026-04-01-morning.md
    │   └── 2026-04-01-debate.md
    └── materials/           # 原始材料
        ├── discord/
        ├── wechat/
        └── twitter/
```

## 6. 语音基础设施（ListenHub 集成）

### 6.1 API 依赖

| 功能 | ListenHub API | 说明 |
|------|--------------|------|
| Voice Clone | Clone API | 上传音频样本，获取 speakerId |
| TTS | TTS API | 用克隆声音生成语音 |
| Multi-Character | TTS Script API | 多角色对话（内心辩论用） |
| ASR | ASR API | 语音转文字（可选，语音输入用） |
| Content Parser | Parser API | URL 内容提取（原材料采集用） |

### 6.2 语音配置

```yaml
# config.yaml
voice:
  provider: listenhub
  speaker_id: "kenny-zh"          # 克隆后的声音 ID
  language: zh                     # 默认语言
  speed: 1.0                       # 语速
  quality: high                    # 音质
  clone_samples:                   # 克隆样本路径
    - materials/voice/sample1.mp3
    - materials/voice/sample2.mp3
```

### 6.3 语音输出策略

不是所有内容都需要语音。按场景决定：

| 场景 | 语音输出 | 说明 |
|------|---------|------|
| 晨间独白 | ✅ 关键段落 | 今日觉察的核心反思 |
| 情绪旁观者 | ✅ 核心回放 | 认知解离的关键句子 |
| 内心辩论 | ✅ 全程 | 辩论本身的核心体验就是听 |
| 时间胶囊 | ✅ 信件全文 | 沉浸式体验 |
| 纯文字对话 | ❌ | 用户打字时只回文字 |
| 进化/修正 | ❌ | 技术操作，不需要语音 |

## 8. 安全边界

1. **仅用于个人成长**，不用于操控、欺骗或冒充他人
2. **数据仅本地存储**，Persona 文件和对话记录不上传任何第三方（ListenHub API 除外）
3. **不鼓励自恋或沉迷**：如果用户表现出过度依赖 AI 自我对话，温和提醒
4. **不替代专业心理治疗**：如果用户表现出严重心理问题，建议寻求专业帮助
5. **隐私保护**：所有数据仅用户本人可访问
6. **Layer 0 硬规则**：AI 不会说出用户绝不可能说的话，除非有原材料证据支持

## 10. 与 ex-skill / colleague-skill 的对比

| 维度 | ex-skill | colleague-skill | **self-distiller** |
|------|---------|----------------|-------------------|
| 蒸馏对象 | 前任 | 同事 | **你自己** |
| 目的 | 怀念/疗愈 | 工作效率 | **自我反思/成长** |
| 互动模式 | 跟 ta 聊天 | 模拟 ta 的反馈 | **跟自己对话** |
| 声音 | 不需要 | 不需要 | **自己的声音（核心）** |
| 价值 | 情感价值 | 效率价值 | **认知价值** |
| 心理学基础 | 怀旧疗法 | 组织行为学 | **认知解离 + 自我觉察** |
| 差异化 | 情感连接 | 工作模拟 | **听到自己想法的冲击力** |

### 为什么 self-distiller 有潜力成为爆款

1. **声音是杀手锏** — ex/colleague 都是纯文字，加上自己的声音后体验质变
2. **心理学背书** — 有真实研究支持"自己的声音增强自我认知"
3. **用户粘性** — 越用越准（Persona 持续进化），形成正循环
4. **传播性** — "跟自己对话"这个概念本身就有话题性
5. **低门槛启动** — OpenClaw 已有数据可以直接用，不需要额外导入
